# Novel De-AI Writing

`novel-de-ai-writing` 是一个面向中文网文生成与改稿的 Skill，用来让模型输出更贴近平台风格、读者口味和商业网文节奏，同时尽量减少明显的 AI 味。

## 这个 Skill 能做什么

- 按不同平台的读者预期调整提示词
- 压低常见 AI 痕迹，比如总结腔、解释腔、过度工整的对白
- 强化场景压力、读者承诺和人物说话差异
- 把市场热点结论转成 Hermes Agent 可直接执行的写作约束
- 补充“通俗可读、少修饰、对白有效、少掉书袋”的大众网文写法约束
- 给 Hermes 明确哪些“去 AI 味偏方”该吸收，哪些要谨慎使用

## 文件结构

- `SKILL.md`：写作与修订流程说明
- `references/platform-prompting.md`：按平台区分的提示词策略
- `references/anti-ai-checklist.md`：降低 AI 味的修订检查表
- `agents/openai.yaml`：Skill 展示元数据

## 建议输入

这个 Skill 最好在 Hermes 已经拿到以下信息后再使用：

- 目标平台
- 目标读者群
- 当前热点套路和卖点
- 前 20 章承诺
- 来自市场分析的反套路提醒

## 适合怎样配合 Hermes Agent

建议在 `novel-market-intel` 之后使用。Hermes 先读取市场简报，再一次只生成一个场景或一个章节开头，最后按 anti-AI 清单做一轮修订后再输出。

## 推荐工作流

1. 先读取 `novel-market-intel` 给出的平台热点和赛道结论。
2. 按平台约束生成单场景或单章开头。
3. 用 anti-AI 检查表做一次修订。
4. 输出给后续评估、续写或章节复盘流程。

## 新增写作取向

这个 Skill 现在额外强调：

- 句子层面追求“大众可读”，不是故意写得粗糙
- 少用空转的环境描写、比喻链和形容词堆叠
- 对话要推动剧情、冲突、信息差或情绪变化
- 可以口语化，但不要机械模仿“底层口吻”
- 脏话、热梗、俏皮话只能在人物和场景真的需要时出现
